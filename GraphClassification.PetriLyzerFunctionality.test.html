getTestElements()¶

Gets the test elements for the certification test cases.

Returns:

temp (ExtendedGDL): The extended GDL for the test case. program1 (GDL): The first GDL program for the test case. program2 (GDL): The second GDL program for the test case. program3 (GDL): The third GDL program for the test case. program4 (GDL): The fourth GDL program for the test case. temp2 (ExtendedGDL): The second extended GDL for the test case. program5 (GDL): The fifth GDL program for the test case.

test_deplete_interval()¶

Tests the depleteInterval method of ExtendedGDL.

It checks if the depleteInterval method can correctly deplete a given interval from a set of intervals.

The test case depletes a set of intervals five times, and checks if the depleteInterval method correctly returns the remaining intervals.

test_edge_mapping()¶

Tests the getEdgeMapping method of ExtendedGDL.

It checks if the getEdgeMapping method can correctly create a mapping between edges of two GDL programs.

test_extension()¶

Tests the extend method of ExtendedGDL.

It checks if the extend method can correctly extend a given ExtendedGDL program with a GDL program and a given mapping.

The test case extends a ExtendedGDL program with a GDL program five times, and checks if the extend method correctly returns True or False.

test_mapping()¶

Tests the map method of ExtendedGDL.

It checks if the map method can correctly create mappings between nodes of two GDL programs.

test_overlap()¶

Tests the overlap method of ExtendedGDL.

It checks if the overlap method can correctly determine if two GDL programs overlap.

The test case overlaps a GDL program with a ExtendedGDL program five times, and checks if the overlap method correctly returns True or False.

test_preconditions()¶

Tests the evaluate method of ExtendedGDL.

It checks if the evaluate method can correctly determine if a given ExtendedGDL program evaluates to True or False with a given set of GDL programs and a given mapping.

The test case evaluates an ExtendedGDL program with a set of GDL programs and a given mapping five times, and checks if the evaluate method correctly returns True or False.

test_prepare()¶

Tests the prepare method of ExtendedGDL.

It checks if the node intervals and edge intervals are correctly set after calling the prepare method.

test_evaluate_lines()¶

Test that the evaluateLine function correctly evaluates the tokens, variables, and numbers of the following text “edge (x1,x2) <.,(-2.5,3.5)>

node x1 <(1,2)>”.

The expected output is - for the first line, the type is 1 and the intervals are (-2.5,3.5) - for the second line, the type is 0 and the intervals are (1,2)

test_lexer()¶

Test that the lexer correctly identifies the tokens, variables, and numbers in the following text “edge (x1,x2) <.,(-2.5,3.5)>

target graph”.

The expected output is - tokens: [‘EDGE’, ‘PL’, ‘VAR’, ‘COMMA’, ‘VAR’, ‘PR’, ‘LESS’, ‘DOT’, ‘COMMA’, ‘PL’, ‘FLOAT’, ‘COMMA’, ‘FLOAT’, ‘PR’, ‘GREATER’, ‘TARGET’] - var: [‘x1’, ‘x2’] - num: [-2.5, 3.5]

test_parser()¶

node x1 <(1,2)> target graph”

into a GDL program.

test_split_lines()¶

node x1 <(1,2)> target graph”.

[‘NODE’, ‘VAR’, ‘LESS’, ‘PL’, ‘FLOAT’, ‘COMMA’, ‘FLOAT’, ‘PR’, ‘GREATER’], [‘TARGET’]

  • var: [‘x1’, ‘x2’], [‘x1’]
