GraphClassification.PL4XGLFunctionality.connected.separate_graph(graph, A)¶

Separates a given graph into connected subgraphs.

This function takes a graph and a list of edges A and returns a list of subgraphs, where each subgraph is represented as a tuple of a list of nodes and a list of edges. The function first creates a copy of the nodes and iteratively removes a node and all of its neighbors from the copy of the nodes, storing the removed nodes and edges in a subgraph. The function continues until all nodes have been removed from the copy of the nodes, at which point it returns the list of subgraphs.

Parameters:

  • graph (Graph) – The graph to be separated.

  • A (list) – The list of edges.

Returns:

A list of connected subgraphs.

Return type:

list

GraphClassification.PL4XGLFunctionality.language.concrete_edge_belong_edge_var(edge_var, edge, X_edge)¶

Checks if an edge belongs to an edge variable constraint.

This function takes an edge variable constraint, an edge, and a dictionary of edge attributes, and returns a boolean indicating whether the edge belongs to the constraint.

The edge variable constraint is a tuple of three elements: the first element is a dictionary of feature indices to intervals of the form (bot, top) over which the feature should be in, the second element is the source node index, and the third element is the target node index. The function iterates over the edge variable constraint and checks if the attributes of the edge satisfy the constraint. If all of them do, the function returns True; otherwise, it returns False.

GraphClassification.PL4XGLFunctionality.language.eval_GDL_program_DFS(GDL_program, graph, data)¶

Evaluates a GDL program over a graph using a depth-first search.

This function takes in a GDL program, a graph, and a data object, and returns a boolean indicating whether the graph satisfies the GDL program.

The function does a depth-first search over the graph, starting from the first edge variable constraint in the GDL program. For each edge that satisfies the constraint, it creates a subgraph and a set of node and edge variable assignments that are consistent with the edge, and then recursively calls itself on the subgraph and the sub-GDL program. If any of the recursive calls return True, the function returns True; otherwise, it returns False.

Parameters:

  • GDL_program (GDLProgram) – The GDL program to evaluate.

  • graph (Graph) – The graph to evaluate the GDL program over.

  • data (Data) – The data object.

Returns:

A boolean indicating whether the graph satisfies the GDL program.

Return type:

bool

GraphClassification.PL4XGLFunctionality.language.eval_GDL_program_on_graphs_GC_Score(GDL_program, data, labeled_graphs)¶

Evaluates a GDL program over a set of graphs in a Data object and returns a score value indicating how well the program matches the labeled graphs.

This function takes a GDL program, a Data object, and a set of labeled graph indices as input and returns a score value between 0 and 1 indicating how well the program matches the labeled graphs.

The function iterates over the graphs in the Data object and checks if the graph satisfies the GDL program. If it does, the graph index is added to either the correct set or the incorrect set. The accuracy of the program is then computed as the ratio of the correct set to the sum of the sizes of the correct set and the incorrect set, and the result is returned as the score.

Parameters:

  • GDL_program (GDL) – The GDL program to evaluate.

  • data (Data) – The Data object containing the graphs.

  • labeled_graphs (set of int) – The set of labeled graph indices.

Returns:

The score value indicating how well the program matches the labeled graphs.

float

GraphClassification.PL4XGLFunctionality.language.find_subgraph_DFS(subgraph, sub_GDL_program, GDL_program, graph, edge_var_idx, node_var_idx_to_concrete_node, edge_var_idx_to_concrete_edge, my_maps)¶

A helper function for find_matching_subgraph.

This function takes in a subgraph, a sub-GDL program, a GDL program, a graph, an index, a dictionary mapping node variable indices to concrete nodes, a dictionary mapping edge variable indices to concrete edges, and a data object.

The function performs a depth-first search over the graph, starting from the first edge variable constraint in the GDL program. For each edge that satisfies the constraint, it creates a subgraph and a set of node and edge variable assignments that are consistent with the edge, and then recursively calls itself on the subgraph and the sub-GDL program. If any of the recursive calls return a matching subgraph, the function returns the matching subgraph; otherwise, it returns None.

Parameters:

  • subgraph (Graph) – The subgraph to search in.

  • sub_GDL_program (GDLProgram) – The sub-GDL program to search for.

  • GDL_program (GDLProgram) – The GDL program to search for.

  • graph (Graph) – The graph to search in.

  • edge_var_idx (int) – The index of the edge variable to search for.

  • node_var_idx_to_concrete_node (dict) – A dictionary mapping node variable indices to concrete nodes.

  • edge_var_idx_to_concrete_edge (dict) – A dictionary mapping edge variable indices to concrete edges.

  • my_maps (Data) – The data object.

Returns:

A matching subgraph, or None if no such subgraph exists.

Return type:

Graph or None
